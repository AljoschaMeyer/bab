WILLIAM3 is a Bab instantiation that is almost identical to BLAKE3. There are only two differences: WILLIAM3 does not supply chunk indices into the label computation for chunks, and WILLIAM3 incorporates a length value into the label computation of inner tree vertices. WILLIAM3 has a normal hash mode and a keyed hash mode (based on a 256 bit key); unlike BLAKE3 it does not support a key derivation mode, and it does not allow for extendable output.

WILLIAM3 uses a chunk_size of 1024 bytes (just like BLAKE3). Its width is 32 bytes (just like BLAKE3).

The hash_chunk function is almost identical to the BLAKE3 computation of chunk chaining values, with a single exceptionExplained in Section 3.5.: where BLAKE3 sets the t parameter of its compression function to the chunk index, WILLIAM3 sets it to 0.

The hash_inner function is almost identical to the BLAKE3 computation of parent node chaining values, with a single exceptionExplained in Section 3.3.: whereas BLAKE3 sets the t parameter of its compression function to 0, WILLIAM3 sets t to the third argument (the length value) of hash_chunk (as an unsigned little-endian 64-bit integer).
